2.1 CURRENT CALIBRATION APPROACH (As of April 2026)
───────────────────────────────────────────────────

The current model uses a **parameter sweep + manual fitting** approach:

A. Fixed Parameters (not optimized)
   K = 3.5 (or 0.54 with DT=0.1)        [matching rate constant]
   π = 1.0 normalized                    [productivity density]
   θ = 0 (instantaneous firing)          [firing threshold]

B. Hand-Tuned Parameters
   s = 0.01                              [separation rate, per month]
   c_max = 0.08                          [max firm sensitivity to GDP]
   c_exponent = 2.0                      [power-law exponent for C distribution]
   zero_fraction = 0.25                  [fraction of GDP-insensitive firms]
   σ_η = 0.01 (idiosyncratic std)        [firm-level demand noise]

C. Empirical Target Moments (fit_report_updated.txt, Jan 2001–Dec 2018)
   u_mean = 0.061                        [target]
   u_std = 0.018
   v_mean = 0.031
   v_std = 0.007
   corr(u,v) = -0.747                    [Beveridge correlation]
   skew(u) = +0.816
   skew(Δu) = +0.447                     [monthly change skewness]
   skew(v) = +0.455

D. Current Model Performance (fit_report_updated.txt)
   u_mean = 0.058 ✓ (within 5% of target)
   u_std = 0.012 ✓ (improved from 0.005)
   v_mean = 0.035 ~ (6% above target; was 0.055)
   v_std = 0.011 ~ (moderate)
   corr(u,v) = -0.517 ~ (too weak; target -0.747)
   skew(u) = +0.951 ✓ (17% above target; much improved)
   skew(Δu) = +0.445 ✓✓ (nearly exact)
   skew(v) = -0.613 ✗ (wrong sign; target +0.455)
   skew(Δv) = -1.283 ✗ (wrong sign; target +0.082)

E. Calibration Method (Phases 1–5 per fitting_plan.txt)
   Phase 1: Load empirical u, v, G and align to monthly grid
   Phase 2: Run simulator with candidate parameters
   Phase 3: Compute loss = MSE(u_sim, u_emp) + MSE(v_sim, v_emp) + λ·corr_loss
   Phase 4: Optimize parameters using scipy.optimize.minimize or Bayesian opt.
   Phase 5: Generate side-by-side empirical vs simulated plots

F. Outstanding Calibration Issues (fit_report_updated.txt, Section 4)
   Issue 1: Vacancy skewness (wrong sign) → may need bilinear matching
   Issue 2: Beveridge correlation too weak → GDP signal amplitude tuning
   Issue 3: Unemployment lower bound too low → structural floor or c_max reduction
   Issue 4: Matching rate K couples vacancy level and recovery speed
            (high K ⟹ correct vacancies but too-fast recovery)


2.2 RECOMMENDED FULL CALIBRATION WORKFLOW (Not yet implemented)
────────────────────────────────────────────────────────────────

Per fitting_plan.txt, a complete automated calibration would consist of:

Phase 1: Empirical Data Preparation ✓ (partially done)
   - Align u(t), v(t), G(t) to monthly frequency
   - Compute summary statistics (mean, std, correlation, skewness, AR(1))
   - Split into train (Jan 2001–Dec 2014) and validation (2015–2018) windows

Phase 2: Lightweight Simulator ✓ (exists in firm.py, economy.py)
   - Takes arbitrary time grid, GDP signal, parameters
   - Returns u(t), v(t), employment trajectories
   - Mirrors src/beveridge/firm.py logic

Phase 3: Loss Function ✗ (exists partially; could be enriched)
   Current: MSE(u) + MSE(v) + λ·corr_loss
   Should add: skew(u), skew(Δu), skew(v), skew(Δv) terms
   Formula:
     L = w_u · MSE(u) + w_v · MSE(v) + w_corr · corr_loss
         + w_skew_u · |skew(u_sim) − skew(u_emp)|
         + w_skew_du · |skew(Δu_sim) − skew(Δu_emp)|
         + w_skew_v · |skew(v_sim) − skew(v_emp)|
         + w_skew_dv · |skew(Δv_sim) − skew(Δv_emp)|

Phase 4: Optimization ✗ (not yet automated)
   Parameters to optimize: {s, c_max, c_exponent, zero_fraction, firing_threshold}
   Bounds:
     s ∈ [0.003, 0.02]      [separation rate; current default 0.01]
     c_max ∈ [0.05, 0.20]   [max sensitivity; current 0.08]
     c_exponent ∈ [1.0, 6.0][power-law shape; current 2.0]
     zero_fraction ∈ [0.05, 0.40] [insensitive firm fraction; current 0.25]
     firing_threshold ∈ [0.0, 0.20] [overstaffing tolerance; current 0.0]
   Method: scipy.optimize.minimize (L-BFGS-B) or differential_evolution
   Runs: 100–200 simulator evaluations (~1–2 hours on single core)

Phase 5: Validation & Plotting ✗ (not fully automated)
   - Generate side-by-side empirical vs simulated plots
   - Compute out-of-sample error on validation window
   - Report optimal parameters and final loss

Phase 6: Multi-Sector Extension ✗ (data exists, not integrated)
   - Disaggregate by 8 major industry sectors
   - Estimate sector-specific C distributions
   - Compare sector-level Beveridge curves

6.1 Parameter Optimization Decision Matrix
────────────────────────────────────────────

Below is a guide for choosing among implementation options:

Decision: Which optimizer to use?
┌─────────────────────────────────────────────────────────────────┐
│ Option A: scipy.optimize.minimize (L-BFGS-B)                    │
│   Pros: Fast, converges to local minimum, widely tested         │
│   Cons: Can get stuck in local minima; no automatic restarts    │
│   Recommended for: Incremental improvements to current fit      │
│                                                                  │
│ Option B: scipy.optimize.differential_evolution                 │
│   Pros: Global optimization, handles discontinuities            │
│   Cons: Slower (100–500 evaluations); less control              │
│   Recommended for: First-time full calibration (Gaps 5, 6, 7)  │
│                                                                  │
│ Option C: Bayesian optimization (scipy.optimize.minimize with  │
│           skopt.gp_minimize wrapper)                            │
│   Pros: Efficient, learns gradient structure                    │
│   Cons: More complex; requires tuning acquisition function      │
│   Recommended for: Fine-tuning after initial estimate           │
│                                                                  │
│ Recommendation: Use differential_evolution for initial calib-   │
│ ration (Gaps 5, 6, 7), then L-BFGS-B for fine-tuning.         │
└─────────────────────────────────────────────────────────────────┘

Decision: Which loss function weights to use?
Current weight vector (if we add skewness terms):
  w_u = 1.0         [unemployment level MSE]
  w_v = 1.0         [vacancy level MSE]
  w_corr = 0.5      [Beveridge correlation penalty]
  w_skew_u = 0.1    [unemployment skewness]
  w_skew_du = 0.3   [unemployment change skewness] ← most important
  w_skew_v = 0.1    [vacancy skewness]
  w_skew_dv = 0.3   [vacancy change skewness]

Rationale:
  - skew(Δu) is nearly exact in current model (+0.445 vs target +0.447)
    so weight it high to keep it there
  - skew(v) and skew(Δv) have wrong sign; lower weights so we don't
    over-constrain other objectives

Sensitivity analysis would help refine these.


6.2 Data Quality Notes
──────────────────────

Unemployment (CPS u_obs):
  - Seasonally adjusted; consistent series since 1948
  - Known issues: misses gig/informal workers (increasing over time)
  - In our sample (2001–2018): largely stable methodology

Vacancies (JOLTS v_obs):
  - Seasonally adjusted; consistent since Dec 2000
  - Known issue: early data more volatile (smaller sample)
  - In our sample: stabilizes after ~2005

GDP (NIPA, BEA):
  - Quarterly, chained 2012 dollars
  - Revisions: typically ±0.1% per quarter
  - Our method: 3-month log-diff minimizes end-point bias vs HP filter

Separation Rate (JOLTS TSR):
  - Total Separation Rate ≈ Quits + Involuntary separations + "other"
  - Quits only ≈ 60% of TSR (cf. Bureau of Labor Statistics)
  - Our s = 0.01/month ≈ 12%/year is reasonable for total separations


6.3 Known Limitations
──────────────────────

1. Model assumes linear production function: output = π · e
   - Reality: production has capital, other inputs
   - Impact: Estimated C may confound capital and labor responses

2. Matching function is linear in u: m = K·u
   - Antoine may have used Cobb-Douglas or other form
   - Bilinear variant (m = K·u·v) is a natural extension

3. Separations are exogenous: s = constant
   - Reality: separations spike in recessions, decline in booms
   - Impact: Model may understate volatility in downturns

4. No wage dynamics
   - Reality: wages adjust, especially in tight labor markets
   - Impact: Model treats labor as single homogeneous pool

5. No capital investment or factor substitution
   - Firms cannot substitute labor for capital or vice versa
   - Impact: Limits realism but simplifies calibration

6. No search frictions at worker level
   - Matching is aggregate only; no individual job search costs
   - Impact: May overstate matching efficiency
